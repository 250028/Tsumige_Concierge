## 1. 調査・相談で使ったプロンプト

### Step1：文脈の共有（最初の1回）

```
# 前提
作るアプリの企画書

（企画書1_250028.md の内容をそのまま貼り付け）
```

> 以降はアプリ名や目的を毎回書かなくて済むようにした。

---

### Step2：レイアウト調査

```
# 調査
この企画書を見て、このジャンルのアプリでよくあるメイン画面の
レイアウトを2～3個、特徴つきで。

# 相談
メイン画面に置く要素とおすすめレイアウトを箇条書きで。
コードはまだ不要。
```

→ AIが3つのレイアウト案（A: ダッシュボード型 / B: 今日の1本+チャット型 / C: カードスワイプ型）をビジュアル付きで提示してくれた。

---

### Step3：3つのいいとこどり

```
この3つのいいとこどりでもう一度
```

→ 短い指示でも前の文脈が引き継がれているので、A・B・C の要素を統合したスマホモックを出力してくれた。

---

### Step4：スマホ vs PC の比較相談

```
# 相談
スマホ想定の方がいい？
PC想定だとどんな感じ？
```

→ スマホ版とPC版（3カラムレイアウト）を並べて比較してもらい、それぞれのメリット・デメリットを整理した。

---

### Step5：方針決定 → design.md 作成依頼

```
スマホファーストで作り、レスポンシブでPC対応で行こうと思う
`design.md` にまとめて
```

→ 会話の流れで決まった設計内容（画面一覧・画面遷移・レスポンシブ方針・テーブル設計）を1ファイルにまとめてもらった。

## 3. AIの出力に自分が加えた変更

- **ゲームタイトルのデータ**：AIが生成したダミーゲームタイトルを、実際に自分が積んでいるゲーム（ゼルダTotK / モンハンワイルズ / ペルソナ5 / スプラ3 / あつ森 / Elden Ring）に差し替えた
- **カラーパレット**：最初の出力がデフォルトBootstrapブルーに寄っていたため、AIに「紫系（#7C3AED）とゴールド（#D97706）を使ってゲームっぽい雰囲気に」と追加指示して変更した
- **チャットの初期メッセージ**：AIが汎用的な例文を入れていたので、コンシェルジュのペルソナ（執事）らしい口調（「ご帰還なさいませ」）に書き直した
- **積みゲー城タブ**：AIのデザイン案にはなかった「達成済み / 未達成」の実績グリッドを自分で追加した
- **app.js のランダム関数**：AIは1つのボタンしか実装していなかったが、スマホ用・PC用の2ボタンに対応するように `randomGame()` を関数化して共通利用できるよう修正した

## 4. AIを使ってみて気づいたこと

**うまくいった指示**

- 最初に企画書を丸ごと貼って文脈を渡したことで、その後の指示が短くても意図が伝わった。「この3つのいいとこどりでもう一度」という短い指示だけで統合されたモックが出てきたのは驚いた。
- 「コードはまだ不要」と明示したことで、余計なコードが出てこず相談に集中できた。設計が固まってからコードを依頼するフローが効率よかった。
- 「スマホ想定の方がいい？PC想定だとどんな感じ？」のように2択で比較を頼むと、メリット・デメリットをセットで出してくれるので意思決定しやすかった。

**うまくいかなかった指示・気づいた注意点**

- 最初の調査プロンプトで「企画書を見て」とだけ書いたら、企画書が共有されていなかったため空振りした。文脈の共有は必ずセットにする必要があることを学んだ。
- AIが生成したゲームタイトルは架空のものが混じっていた。実在のゲーム名を使いたい場合は自分でデータを書き換える必要がある（ハルシネーション対策として自分でチェックする習慣が大事だとわかった）。
- カラーの指定は「ゲームっぽい」などの曖昧な表現より、「#7C3AED の紫を primary に」と具体的なHEX値で伝えた方が意図通りの出力が得やすかった。
